# Хранение Истории матчей: файл-на-матч (JSON) + полный журнал, не SwiftData

История матчей хранит каждый сыгранный матч как **отдельный JSON-файл** в каталоге `history/`
(имя с префиксом-таймштампом `endedAt` для дешёвой сортировки), а не как записи SwiftData/Core Data
и не как один общий файл-массив. В каждой записи держим **полный журнал очков** (`events`), а не только
итоговый счёт. Запись — это `MatchRecord`-обёртка (`id`, `startedAt`, `endedAt`, `settings`, `events`,
`workout?`) над *чистым* `ScoringEngine`, который про даты/id/тренировку ничего не знает.

## Контекст и трейд-офф

- **Почему не SwiftData (нативный путь).** SwiftData даёт запросы/сортировку «из коробки» (пригодилось бы
  для будущих агрегатов), но тащит схему, миграции и привязку к фреймворку. У нас `ScoringEngine` уже
  `Codable`, синк планируется в **свой бэкенд**, а не в iCloud, и нужды пока — список + просмотр. Для этого
  SwiftData — оверкилл. Переход к нему оправдан позже, если агрегированная статистика станет тяжёлой.
- **Почему файл-на-матч, а не общий массив.** Битая запись теряет один матч, а не всю Историю; добавление
  матча — создание одного файла без перезаписи общего; на каждый матч удобно навесить состояние синка позже.
- **Почему полный журнал, а не итог.** Из журнала восстановим любой будущий показатель и покадровый реплей
  без миграции данных. Это согласуется с event-sourcing-движком.
- **Почему обёртка, а не поля в движке.** `ScoringEngine` поедет в TS-веб как есть; даты/id/тренировка —
  забота слоя приложения и персистентности.

## Следствия

- Текущий матч (`current-match.json`) тоже становится `MatchRecord` (active: `endedAt == nil`), чтобы `id`
  и `startedAt` существовали с момента старта — это даёт идемпотентный коммит и дедуп при будущем синке.
- Статус матча (completed/abandoned) **не** хранится полем — производится из журнала (`state.isFinished`).
- Синк пока не строим, но запись держим sync-ready: стабильный `UUID` + чистый `Codable`, без `synced`-флага
  (добавится тривиально, формат файл-на-матч это позволяет).
