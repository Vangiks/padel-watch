# День на облачном Mac (Macly) — чек-лист

Цель дня: **впервые собрать приложение, увидеть его в симуляторе Apple Watch, поймать правки
компилятора и проверить вид/навигацию каждой фичи.** Логика счёта уже покрыта тестами —
здесь проверяем именно UI и сборку.

Работай сверху вниз. На каждом «спотыкании» — копируй точный текст ошибки и шли мне.

## 0. Перед стартом (5 мин)
- [ ] Открой **Xcode** → согласись с лицензией, дождись установки компонентов.
- [ ] Проверь, что установлен **watchOS Simulator**: Xcode → Settings → Components/Platforms →
      watchOS должен быть скачан. Если нет — поставь (это может занять время, делай первым).
- [ ] Установи XcodeGen:
      ```bash
      brew install xcodegen
      ```

## 1. Достать код (приватный репозиторий)
На свежей VM ключей нет — проще всего через GitHub CLI:
```bash
brew install gh
gh auth login          # выбери GitHub.com → HTTPS → авторизуйся в браузере
gh repo clone Vangiks/padel-watch
cd padel-watch
```
> Альтернатива: сгенерировать SSH-ключ (`ssh-keygen -t ed25519`), добавить `.pub` в
> GitHub → Settings → SSH keys, затем `git clone git@github.com:Vangiks/padel-watch.git`.

## 2. Прогнать тесты логики (быстрая проверка, что всё на месте)
```bash
swift test --package-path ScoringEngine
```
Ожидаем: **23 теста, 0 ошибок** (как на Linux).

## 3. Сгенерировать и открыть проект
```bash
xcodegen generate
open PadelWatch.xcodeproj
```

## 4. Запуск в симуляторе
- [ ] В тулбаре Xcode выбери схему **PadelWatchApp** и симулятор **Apple Watch** (любой свежий).
- [ ] Нажми ▶ (Run).

### Если ругается на подпись (HealthKit-entitlement требует команду)
Для симулятора подпись не нужна, но capability HealthKit иногда заставляет выбрать team.
Любой из вариантов:
- В таргете → Signing & Capabilities → включи **Automatically manage signing** и выбери свой
  **Personal Team** (бесплатный Apple ID), **или**
- Временно убери строку `CODE_SIGN_ENTITLEMENTS` из `project.yml`, `xcodegen generate`, и
  запусти без HealthKit (вид проверишь и так; активность всё равно невидима в симуляторе).

## 5. Что проверить визуально (по фичам)

**Мастер настройки**
- [ ] Стартовый экран: «Падел», секция «Формат» (Классика/Турнир).
- [ ] Классика → экран «Режим ровно» c 4 пунктами: Больше/меньше · Золотой мяч · Золотой ×2 · Star Point.
- [ ] Далее «Сеты» (1/3) → «Первая подача» (Подаю я / Подаёт соперник) → открывается счёт.
- [ ] Турнир → экран «Очки»: пресеты 16/24/32 + кручение Digital Crown меняет число.

**Экран счёта (классика)**
- [ ] Тап по левой зоне: 0→15→30→40→гейм (число слева растёт, потом сбрасывается, гейм +1).
- [ ] Верхняя строка «Сеты X-Y / Геймы X-Y», значок 🎾 у подающей стороны, меняется по геймам.
- [ ] Доведи до 40-40 → у повёдшего показывается **AD**.
- [ ] В режиме «Золотой мяч»: на первом 40-40 числа подсвечиваются жёлтым (решающее очко).
- [ ] Кнопки внизу: **↩︎ undo** (откатывает), **⏸ пауза** (оверлей «Пауза» + «Продолжить»),
      **↪︎ redo** (возвращает откатанное).

**Быстрый прогон до конца матча** (чтобы увидеть экран итога без 6 геймов):
- [ ] Заведи **Турнир**, Digital Crown'ом скрути N до **4**, старт → набей 4 очка →
      экран итога: победитель/ничья + счёт. (В классике быстрее всего — 1 сет, но это 6 геймов.)

**Восстановление матча**
- [ ] В середине матча сделай Stop в Xcode (имитация закрытия) → снова Run →
      на стартовом экране сверху появляется «Продолжить матч» с текущим счётом.

**Локализация**
- [ ] Симулятор → Settings (Watch) → General → Language → English (или схема Run →
      Options → App Language → English) → перезапуск: подписи становятся английскими
      (Format, Classic, Sets, New match, …).

**Активность (ограниченно в симуляторе)**
- [ ] При старте матча — системный запрос доступа к Health (один раз). Принять.
- [ ] Живого пульса/калорий в симуляторе не будет — проверяем только что **нет краша**.
      Реальные данные — позже на физических часах.

## 6. Вероятные места правок компилятора
Если что-то не соберётся — смотри сюда в первую очередь, это самые «свежие» и нетривиальные куски:
- `WorkoutManager.swift` — изоляция делегатов HealthKit (`nonisolated` + `Task { @MainActor }`),
  доступность API (`HKQuantityType(.heartRate)` требует watchOS 8+ — у нас 10, ок).
- `@Observable` + `@State` связки в `RootView`/`MatchViewModel`/`AppCoordinator`
  (Observation, watchOS 10+).
- `String(localized:)` с интерполяцией — ключи должны совпасть с каталогом; если Xcode
  подсветит «stale/missing» в `Localizable.xcstrings` — это не ошибка сборки, поправим в редакторе каталога.
