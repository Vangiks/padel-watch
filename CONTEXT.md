# Context — Vamos (padel-watch)

Глоссарий доменного языка проекта. Только термины и их значения — без деталей реализации.

## Standalone-runtime

Свойство работающего приложения: **работающее приложение не требует ни сети, ни присутствия iPhone поблизости.** Матч можно вести на «голых» Apple Watch на корте.

Разводим три перегруженных смысла «offline»:

- **Runtime (исполнение)** — приложение не делает ни одного сетевого запроса. Это и есть утверждаемая гарантия.
- **Delivery (доставка/установка)** — установка и обновление через TestFlight/App Store, а также первичная настройка часов (требует разово спаренного iPhone) нуждаются в инфраструктуре Apple. Это **вне** домена работы приложения и гарантию не нарушает.
- **Future sync (post-MVP)** — выгрузка результатов в веб-приложение потребует сети, но в MVP отсутствует и держится явно за границей.

Связанные различия:

- **«Нет сети в runtime» ≠ «iPhone не нужен рядом».** Это две отдельные гарантии, обе выполняются: первая — из отсутствия сетевых вызовов; вторая — из standalone-конфигурации (watch-only, без канала связи с телефоном).
- **«Приложение не делает сетевых запросов» ≠ «данные никогда не покидают часы».** Тренировка пишется в локальный store «Здоровья»; дальнейшая синхронизация Health в iPhone/iCloud — это забота watchOS, инициируется не приложением и асинхронна. Утверждение об offline — про поведение приложения, а не privacy-гарантия о неперемещении данных.
